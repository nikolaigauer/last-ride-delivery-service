Don't go silent for long stretches before writing files. User noticed a 1h20m gap with no output.

**Why:** Long internal planning without visible output looks like a loop or freeze, and burns session time the user can't see.

**How to apply:** When a task involves planning before coding, give a 1-sentence status note ("Working on Bridge.js — figuring out ravine coordinates") before diving in. Commit to a design and write it — don't re-plan from scratch in reasoning. Check in with the user after each major file write, especially on multi-file tasks.
